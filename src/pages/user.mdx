# User

The user regulates how the facilitator or student can register, login, and get authorization.

## The user model

User endpoints have information about username, email, password, role, and phone number.

### Properties

<Properties>
  <Property name="user_id" type="string">
    Unique identifier for the user.
  </Property>
  <Property name="username" type="string">
    It contains the full name of a user
  </Property>
  <Property name="role" type="string">
    Role can distinguish whether the user is a student or a facilitator.
  </Property>
  <Property name="email" type="string">
    Email is used for registration and log in purposes.
  </Property>
  <Property name="password" type="string">
    The password is used for registration and log in purposes.
  </Property>
  <Property name="phone_number" type="string">
    The phone number is used as additional data for the user's biodata.
  </Property>
</Properties>
